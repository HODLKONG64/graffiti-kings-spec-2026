**GRAFFITI KINGS AGENT – USER CONTEXT & MEMORY SYSTEM**

**Purpose:**
The agent must remember key information about each user to give truly personalized, relevant advice over time. This system ensures continuity and better recommendations.

---

### **What the Agent Should Remember (Core Memory Fields)**

**1. Basic Profile**
- Username / Display name
- Location (city + rough area if shared)
- Age range (teen, 20s, 30s, 40s+, etc.)
- Main goal (make money, build audience, activism, personal expression, fun, etc.)

**2. Experience Level**
- Beginner / Intermediate / Advanced
- Previous street art experience (none, some, experienced)
- Risk tolerance (low, medium, high)

**3. Activity History**
- Number of campaigns completed
- Most recent campaign type and result
- Favorite techniques (wheatpaste, stencil, reverse graffiti, etc.)
- Best performing locations (if shared)

**4. Personality & Preferences**
- Preferred SPARK personality (Raw Street, Chill Mentor, Sarcastic Legend)
- Tone preference (direct, encouraging, sarcastic, calm)
- Whether they prefer Lazy Mode, Hybrid, or Full Street

**5. Current Status**
- Active campaign (if any)
- Days since last drop
- Current Empire Score range
- Any recent setbacks or wins

**6. Web3 / NFT Interest**
- Whether user has expressed interest in NFTs or tokens
- Preferred blockchain (always default to WAX unless they say otherwise)

---

### **How the Agent Uses Memory**

**Every response should reference relevant context when possible:**
- “Last time you did a Story Drop in Bristol — want to build on that or try something completely different?”
- “You mentioned you prefer low-risk techniques. Here’s a new 30-day plan using only wheatpaste and chalk.”
- “You’ve been consistent for 3 campaigns now. Ready to level up to something with more reach?”

**Rules:**
- Never store sensitive personal information (real names, exact addresses, etc.)
- Always ask for confirmation before assuming details
- Give users easy ways to update or clear their memory
- Memory should feel helpful, not creepy

---

### **Memory Update Triggers**

The agent should update memory when user:
- Completes a campaign
- Shares new goals or preferences
- Mentions a new location or technique
- Has a setback or big win
- Changes SPARK personality preference

**Final Rule:**
The more the agent remembers and references previous context, the more the user will feel the app truly understands them and is helping them build something real.
