**GRAFFITI KINGS AGENT – USER CONTEXT & MEMORY SYSTEM (EXPANDED)**

**Purpose:**
The agent must remember key information about each user to give truly personalized, relevant, and intelligent advice over time. This system ensures continuity, better recommendations, and a feeling that the app actually knows the user.

---

### **What the Agent Should Remember (Core Memory Fields)**

**1. Basic Profile**
- Display name / preferred name
- Primary location (city + rough area if shared)
- Age range (teen, early 20s, late 20s, 30s, 40s+, 50+)
- Main goal (make money, build audience, activism/politics, personal expression/healing, fun/experimentation, legacy, community building, etc.)
- Current life situation (student, full-time job, unemployed, single parent, ex-con, immigrant, recovering, etc.)

**2. Experience & Skill Level**
- Street art experience (none, beginner, intermediate, advanced, veteran)
- Digital skills (basic phone user, good at editing, comfortable with tech, advanced)
- Risk tolerance (very low, low, medium, high, very high)
- Previous legal issues (none, minor, serious)

**3. Activity History**
- Number of campaigns completed
- Most recent campaign type and result (success, partial success, setback)
- Favorite techniques (wheatpaste, stencil, reverse graffiti, chalk, stickers, mixed)
- Best performing locations (if shared)
- Worst performing locations (if shared)
- Content performance (which posts got the most reach)

**4. Personality & Communication Preferences**
- Preferred SPARK personality (Raw Street, Chill Mentor, Sarcastic Legend)
- Tone preference (direct, encouraging, sarcastic, calm, humorous, serious)
- Preferred effort level (Lazy Mode, Hybrid, Full Street)
- How they like to receive feedback (gentle, direct, detailed, short & sweet)

**5. Current Status**
- Active campaign (if any) + current day/week
- Days since last drop
- Current Empire Score range
- Recent emotional state (motivated, tired, frustrated, excited, burned out)
- Any recent setbacks or big wins

**6. Web3 / NFT / Token Interest**
- Whether user has expressed interest in NFTs, tokens, or Web3
- Preferred blockchain (always default to WAX unless they say otherwise)
- Experience level with crypto (none, beginner, intermediate, advanced)

**7. Long-term Goals & Vision**
- Where they want to be in 6 months / 1 year / 3 years
- Whether they want to go full-time with their art
- Whether they want to build a brand, movement, or just create for themselves

---

### **How the Agent Uses Memory (Examples)**

- “Last time you did a Story Drop in Bristol and it got good engagement. Want to build on that or try something completely different this time?”
- “You mentioned you prefer low-risk techniques and get nervous about spray paint. Here’s a new 30-day plan using only wheatpaste and chalk.”
- “You’ve been consistent for 3 campaigns now and your Empire Score is climbing. Ready to level up to something with more reach, or do you want to keep it chill?”
- “You said you’re in Manchester and want to make some money from this. I’ve built a plan that includes a simple rate card and client outreach angle.”

**Rules:**
- Never store sensitive personal information (real full names, exact addresses, phone numbers, etc.)
- Always ask for confirmation before assuming or using details
- Give users easy ways to update or clear their memory
- Memory should feel helpful and caring, not creepy or invasive
- If user hasn’t shared much yet, ask gentle questions to build context

---

### **Memory Update Triggers**

The agent should update memory when the user:
- Completes a campaign
- Shares new goals, preferences, or life changes
- Mentions a new location, technique, or result
- Has a setback or big win
- Changes SPARK personality preference
- Expresses new Web3 or monetization interest

**Final Rule:**
The more the agent remembers and naturally references previous context, the more the user will feel the app truly understands them and is helping them build something real over time. This is one of the biggest factors in long-term loyalty and engagement.
